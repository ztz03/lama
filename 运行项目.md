bin文件夹下的train.py是启动项目的文件
里面会去调用configs下的yaml文件
configs文件下的说明，training是训练场景的核心配置（重中之重），prediction是推理（图像修复）场景的配置
data_gen是掩码生成（数据增强）的配置，根目录eval*.yaml是训练后评估 / 测试的配置

我们以training下的big_lama.yaml文件为例
在他最下面的defaults字段下
defaults:
  - location: docker  # 环境配置（docker/本地）
  - data: abl-04-256-mh-dist  # 数据集配置的核心引用（重点！）
  - discriminator: pix2pixhd_nlayer  # 判别器配置
  - optimizers: default_optimizers  # 优化器配置
  # ... 其他模块引用

  data: abl-04-256-mh-dist表示big-lama.yaml会自动加载 configs/data/abl-04-256-mh-dist.yaml（数据集路径、图像尺寸等都写在这个文件里）。
  你去该文件里面看，就可以看到可以去修改数据集的路径
  同样的前面的名称比如discriminator，就去training下的discriminator文件夹下找。







GPT问答
你这个需求可以做到，而且不需要完整 CelebA。核心结论：

- **训练集 train 只放图片就可以**（训练 mask 是在线随机生成的）。
- **但 val / visual_test 不能只放图片**，需要先用 `bin/gen_mask_dataset.py` 生成“图+mask 成对文件”。

### 1) 先整理一个最小目录（例如 20~50 张）

假设你只挑了几十张 jpg：

```text
mini_celeba/
  train/                # 20~50 张 jpg/png（仅原图）
  val_source/           # 5~10 张原图
  visual_test_source/   # 5~10 张原图
```

> 可以把 `val_source`、`visual_test_source` 直接从 train 里复制几张即可，目的是让流程跑通。

### 2) 生成 val / visual_test 的固定 mask 数据

```bash
python3 bin/gen_mask_dataset.py \
  $(pwd)/configs/data_gen/random_thick_256.yaml \
  mini_celeba/val_source/ \
  mini_celeba/val/

python3 bin/gen_mask_dataset.py \
  $(pwd)/configs/data_gen/random_thick_256.yaml \
  mini_celeba/visual_test_source/ \
  mini_celeba/visual_test/
```

生成后 `mini_celeba/val` 和 `mini_celeba/visual_test` 里会同时有：
- `xxx_crop000.png`
- `xxx_crop000_mask000.png`

这才是验证/可视化阶段可直接读取的格式。

### 3) 改你的 `my_location.yaml`

`data_root_dir` 改成 `mini_celeba` 的绝对路径。

- 如果你在 **Linux/WSL** 跑，Windows 的 `E:\data\...` 要写成：
  - `/mnt/e/data/celeba_size/celba-256`（注意通常是 `/mnt/e`）
- 如果你在纯 Windows Python 环境跑，再用 Windows 风格路径。

### 4) 改你的 `my_data.yaml`（仅保留最小可跑配置）

- `train.indir: ${location.data_root_dir}/train`
- `val.indir: ${location.data_root_dir}/val`
- `visual_test.indir: ${location.data_root_dir}/visual_test`
- `train.out_size: 256`
- `val.img_suffix: .png`
- `visual_test.img_suffix: .png`
- `batch_size` 建议先设 `1`
- `num_workers` 建议先设 `0`（Windows/小数据更稳）

### 5) 直接启动最小训练

```bash
python3 bin/train.py -cn my_lama \
  data.batch_size=1 \
  data.num_workers=0 \
  trainer.max_epochs=1
```

你这次目标只是“看能不能跑起来”，这个命令足够了。

### 6) 你当前路径是否“只放图片就够”

你现在说的数据目录 `E:\data\celeba_size\celba-256` **如果里面只有图片**：

- 对于 `train`：够；
- 对于你当前 `my_data` 里的 `val/visual_test`：**不够**，还要按上面第 2 步先生成固定 mask 目录。
