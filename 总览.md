# saicinpainting 总览（数据集处理、运行流程、掩码机制）

## 0. 你最该先记住的目录结构
`saicinpainting/` 主要分成两大块：

- `training/`：训练时的数据加载、掩码生成、网络模块、损失函数、Trainer。
- `evaluation/`：评估数据集、评估指标（SSIM/LPIPS/FID）与分割掩码相关逻辑。
- `utils.py`：通用工具（ramp、DDP 辅助、日志辅助等）。

---

## 1. 数据集如何处理

### 1.1 训练数据（`training/data/datasets.py`）

#### `InpaintingTrainDataset`
- 从 `indir` 递归读取 `*.jpg`。
- 用 OpenCV 读取并转 RGB。
- 用 albumentations 做图像增强。
- 转成 `C,H,W`。
- 调用 `mask_generator(img, iter_i=...)` 生成掩码。
- 返回：`{'image': img, 'mask': mask}`。

这就是训练时最核心的样本格式。

#### 其他训练数据类
- `InpaintingTrainWebDataset`：支持 webdataset 流式读取数据。
- `ImgSegmentationDataset`：除图像和掩码外，还可加载语义分割标签（one-hot + class id）。

#### 训练 dataloader 组装
`make_default_train_dataloader(...)` 做三件事：
1. 创建 mask generator（`get_mask_generator`）。
2. 创建图像增强 pipeline（`get_transforms`）。
3. 根据 `kind` 选具体 Dataset 并包装成 `DataLoader`；DDP 时挂 `DistributedSampler`。

---

### 1.2 验证/评估数据（`training/data/datasets.py` + `evaluation/data.py`）

#### 验证集创建入口
`make_default_val_dataset(...)` 支持：
- `default`
- `our_eval`
- `img_with_segm`
- `online`

其中 `online` 很关键：会在线生成掩码，而不是依赖预存 mask 文件。

#### 评估数据格式约定（`evaluation/data.py`）
`InpaintingDataset` 默认通过 `*mask*.png` 找掩码，再映射回原图路径（如 `xxx_mask...png -> xxx.jpg`）。

此外支持：
- `scale_factor`：缩放图像/掩码。
- `pad_out_to_modulo`：把尺寸 pad 到指定模数（常用于网络下采样对齐）。

## 2. 掩码是如何生成的

## 2.1 训练时掩码（`training/data/masks.py`）

支持多种生成器：
- **RandomIrregularMaskGenerator**：不规则笔触（线条/圆/方块）。
- **RandomRectangleMaskGenerator**：随机矩形块。
- **RandomSegmentationMaskGenerator**：基于语义分割候选对象生成。
- **RandomSuperresMaskGenerator**：网格型遮挡。
- **OutpaintingMaskGenerator**：在图像边缘方向生成“外扩修复”掩码。
- **DumbAreaMaskGenerator**：简单中心或随机方形遮挡。

最后由 **MixedMaskGenerator** 按概率混合采样，并可通过 `invert_proba` 反转掩码。

对外总入口：`get_mask_generator(kind, kwargs)`，默认 `kind='mixed'`。

---

### 2.2 分割掩码细节（`evaluation/masks/mask.py`）

`SegmentationMask` 的思路：
1. 使用 detectron2 的 panoptic segmentation 得到场景实例。
2. 选择 `isthing=True` 且面积不过大的对象作为候选遮挡对象。
3. 对对象 mask 做下采样、缩放、翻转、位移等增强。
4. 控制与已有掩码/前景对象的交并约束，保证 mask 合理。
5. 返回一组可用 mask（`get_masks`）。

这套策略的核心是：遮挡对象“像真的会被移除的物体”，而非完全随机涂抹。

## 3. 模型训练运行链路（在 `saicinpainting` 包内）

## 3.1 Trainer 创建
- `make_training_model(config)` 读取 `config.training_model.kind`。
- 默认映射到 `DefaultInpaintingTrainingModule`。

## 3.2 Base 模块负责搭框架
`BaseInpaintingTrainingModule` 中会初始化：
- generator / discriminator
- adversarial loss
- visualizer
- evaluator
- train/val dataloader

并提供 `training_step` / `validation_step` 的统一流程。

## 3.3 Default 模块定义 inpainting 核心前向
在 `DefaultInpaintingTrainingModule.forward()`：
1. `masked_img = image * (1 - mask)`。
2. 可选加噪声通道。
3. 可选拼接 mask 通道。
4. 输入 generator 得到 `predicted_image`。
5. 融合得到最终 `inpainted = mask * predicted + (1-mask) * image`。

## 3.4 损失组合
`generator_loss()` 中典型组合：
- masked L1
- perceptual loss
- adversarial loss
- feature matching loss
-（可选）resnet perceptual

---

## 4. 评估与可视化

## 4.1 评估器
`make_evaluator(...)` 可构建包含：
- SSIM
- LPIPS
- FID
并可加综合指标（例如 `ssim_fid100_f1`）。

`InpaintingEvaluatorOnline` 负责：
- 按 batch 累计指标。
- 按 mask 面积分桶统计。
- 在 `evaluation_end()` 统一汇总。

## 4.2 可视化
`DirectoryVisualizer` 会将 `image / predicted_image / inpainted` 拼图后保存到目录，方便跟踪训练效果。

## 5. 你接下来可以怎么读源码（建议顺序）
1. `training/data/datasets.py`：先搞懂样本字典里有哪些 key。
2. `training/data/masks.py`：搞懂 mask 分布从哪里来。
3. `training/trainers/default.py`：搞懂 forward 与 loss 如何消费 `image/mask`。
4. `evaluation/data.py` + `evaluation/evaluator.py`：搞懂验证评估的数据与指标。
5. `evaluation/masks/mask.py`：再深入分割驱动掩码生成。
